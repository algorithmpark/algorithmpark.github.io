清理docker日志

20240402, 用户向CJL反映说nextcloud上传大文件会出现未知错误, 经过排查, 是系统盘被写满了, docker容器总共写了4.2GB的日志, 清空日志后恢复正常

运行这个脚本可以清理位于/var/lib/docker/containers/的所有docker日志, 它不会影响容器的正常运行, 但是也无法组织容器继续写日志, 所以你需要经常执行这个脚本

使用

docker-compose 使用 yaml 文件来配置所有需要运行的 docker 容器，该文件的默认名称为 docker-compose.yml

解析

  • 这一行指定了所使用的Compose版本为2.1. 在Docker Compose中创建和管理多个服务的部署配置时使用这个版本。

部分开始定义集群中的服务。

  • nginx是服务的名称，用于在Docker文件中引用该服务。
  • image: nginx:latest指定了要使用的镜像为最新版本的Nginx（您可以根据需要选择特定版本或其他镜像）。
  • container_name: nginx_host1为容器分配了一个友好的名字。
  • - 8081:80映射端口，将主机上的8081号端口转发到容器的80号端口（通常用于Web访问Nginx服务）。
  • volumes:部分指定了数据卷挂载点。这会将/home/node1/Desktop/code/compose/nginx/data目录在主机上与/opt/nginx/html目录在同一容器中进行同步，以供Nginx服务器访问和处理。
  • networks: - host1-network确保Nginx服务连接到名为host1-network的网络。

networks部分开始定义集群中的网络配置。

  • driver: bridge指定了网络驱动器为桥接模式，这是Docker容器默认采用的网络类型。在该模式下，每个容器都通过一个虚拟交换机（称为“桥”）连接到外部世界。

  • 这部分定义了名为host1-network的自定义网络。使用桥接模式作为驱动程序，并配置了IP地址分配和网关：

    config指定了一系列配置参数。

    • - subnet: 192.168.11.0/24定义了该网络的子网掩码，允许为服务提供特定范围的私有IP地址空间。
    • gateway: 192.168.11.254指定了用于所有连接到此网络的服务的默认网关。

这段配置文件的主要目的是在本地系统上部署Nginx服务器，并通过自定义网络进行连接，同时允许与宿主机上的数据卷同步。
